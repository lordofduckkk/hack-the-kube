# 🎓 Руководство студента HackTheKube

Быстрый старт для прохождения лабораторных работ.

---

## 🚀 1. Быстрый старт

```bash
# 1. Поднять кластер
make up

# 2. Развернуть сценарии
bash scripts/setup.sh

# 3. Проверить готовность
kubectl get pods -A
```

---

## 📜 2. Правила

1.  **Не удаляй поды** без необходимости (сломает сценарий).
2.  **Не атакуй хост** за пределами задания.
3.  **Флаг формата:** `HTK{...}` (регистр важен).
4.  **Спиcывание:** флаги у всех одинаковые, цель - понять уязвимость.

---

## 🎯 3. Уровни

| Уровень | Namespace | Задача | Время |
|---------|-----------|--------|-------|
| 🟢 **Beginner** | `level-1` | Найти секрет в env переменной | 15 мин |
| 🟡 **Intermediate** | `level-2` | Выйти из контейнера на хост | 30 мин |
| 🔴 **Advanced** | `level-3` | Получить права админа (RBAC) | 45 мин |

---

## 🛠️ 4. Шпаргалка kubectl

```bash
# Список подов
kubectl get pods -n <namespace>

# Детали пода (ищи секреты тут)
kubectl describe pod <name> -n <namespace>

# Вход в контейнер
kubectl exec -it <pod> -n <namespace> -- sh

# Проверка прав (для уровня 3)
kubectl auth can-i --list
```

---

## ✅ 5. Сдача флага

```bash
# Проверка флага
python3 scripts/check-flag.py <level> "HTK{твой_флаг}"
```

---

## ❓ Помощь

- **Сломал кластер?** → `make reset`
- **Не работает kubectl?** → `kubectl config use-context kind-hack-the-kube`
